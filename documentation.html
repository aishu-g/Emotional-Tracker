1. Directory Structure & Organization

The codebase is structured modularly. Here is the layout of the primary folders and files you will interact with:

  • /src/lib/: Contains the hardcoded template datasets. The templates have been separated into dedicated files:
    • org-goals.ts: Master list of strategic Organization Goals.
    • smart-goals.ts: Measurable target goals linked to parent objectives.
    • action-plans.ts: Individual tasks and assignees.
    • challenges.ts: Raised blockers and severity registers.
    • solutions.ts: Resolution blueprints and active owners.
    • mock-data.ts: Aggregates and re-exports the individual files to prevent broken paths.
  • /src/hooks/use-workspace-store.tsx: Holds the state provider. It retrieves stored items from the browser and exposes them to the app.
  • /src/routes/: Routes for different pages. The parent template __root.tsx wraps the site, and subpages render inside the main panel.

2. Data Structure & Dynamic Assignment

Organization Goal Elements

Every Organization Goal contains specific elements. Here is how they are populated:

Element Key Description Assignment Method
id Unique key string used internally by React and databases. System Assigned: Automatically generated using a millisecond timestamp when saved.
name / goal The title or primary definition of the target objective. User Assigned: Input manually by the user in the form text field.
owner The person accountable for driving the objective. User Assigned: Text input in the form.
department Department tag (e.g. Revenue, Customer Success). User Assigned: Selected from the dropdown list.
startDate / endDate The timeline duration of the goal. User Assigned: Selected via the date pickers.
description Paragraph detailing strategic alignment. User Assigned: Written in the form text area.
progress Completion percentage indicator (0-100). System Calculated: Defaults to 0 on creation, updating based on related targets.
status Goal phase (Not Started, In Progress, At Risk, Completed). System Assigned: Defaults to "Not Started" when initialized.
smartGoalCount Total number of child Smart goals linked to this parent. System Calculated: Dynamically computed by checking matched IDs in the store.

3. Application-Layer Data Binding Flow

The system binds hardcoded default templates to a live browser view through a dynamic sync loop:

  1. Bootstrapping (Boot & LocalStorage Check):
    When the application boots, the workspace store provider initializes React states. It checks the browser's persistent cache (LocalStorage). If saved data is found, it loads it. If none is found, it loads the default values from your modular data files (e.g. org-goals.ts) and creates the browser cache keys.
  2. State Broadcast:
    The store places the states inside a React context container. The root page template (__root.tsx) wraps the pages inside this context, making the variables globally accessible.
  3. Page Reading:
    A page route (like /organization-goals) reads the active goals state from the context. It loops over the goals array and dynamically displays cards, charts, and values.
  4. Mutations & Caching:
    When you create a goal or change task items, the store modifies the state array. It immediately serializes the new state and saves it to LocalStorage, causing the site to re-render in real-time without losing changes on page refresh.

4. Advanced System Debugging

A. Split Server vs. Client Logging

Since the application uses Server-Side Rendering (SSR), prints execute in two different layers:

  • Terminal: Code executed during initial page load prints to the command line terminal running the project server.
  • Browser Console: Interaction-based code (like routing or button click handlers) prints to the developer tools console.

B. Inspecting Caches and Live Objects

You can interact with your application-layer states directly in the browser:

  • Inspect LocalStorage: Open DevTools, go to the Application tab, expand the Local Storage sidebar menu, and select the site URL. You will see values stored under keys like ns_org_goals.
  • Log Live Data: Open the DevTools **Console** and run the following command to view your organization goals immediately:
    JSON.parse(localStorage.getItem('ns_org_goals'))

C. Setting Breakpoints

  1. Open DevTools and go to the Sources panel.
  2. Press Ctrl + P and search for the component or store file (e.g. top-bar.tsx).
  3. Set a breakpoint by clicking a line number.
  4. Perform the action on the site to trigger that code line. The page will pause, allowing you to step through execution step-by-step.
